Selection

This page explains the Selection use case which is provided with PTV xServer .NET. It shows how geographical elements provided by a layer can be selected for subsequent operations.

This page contains the following sections:

  • Preparations
  • Use case manual
Preparations

Presuming an already existing layer, an instance of SelectInteractor is created, which is responsible of managing a subset of the geographical objects in this layer. Besides the providing of handlers for mouse events (changing the subset of selected objects) and the drawing events (redraw after a change of the subset), the painting order is adjusted via SetZIndex, to guarantee a drawing always on top of all other layers.

C#
Copy
this.map = map;
this.shapes = shapes;

map.MouseLeftButtonUp += map_MouseLeftButtonUp;
map.MouseLeftButtonDown += map_MouseLeftButtonDown;
map.MouseMove += map_MouseMove;

SetZIndex(this, 999999999);
SelectedElements.CollectionChanged += SelectedElements_CollectionChanged;
Use case manual

The Selection use case shows how to select geographical objects in a ShapeLayer. Starting the demo application, a shape layer is created after the checkbox is activated in the Selection expander:

Selection use case

By left-clicking into the map nearby one of the blue triangles, they get be highlighted with a red semi-transparent circle. By dragging via left mouse button and keeping the Alt-Button pressed, a polygon search can be initiated, i.e., all objects inside the dragged polygon will be selected.
