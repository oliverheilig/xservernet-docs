Use the Map Gadgets

PTV offers a set of map gadgets which can additionally be shown in the map. The gadgets supply additional information and functionality which can be used in the map. For example, there is a zoom slider and a scale gadget.

All map gadgets can separately be shown and hidden. The gadgets have a default style, showing them in a dark silver look. This look can be changed to adapt the gadgets so that they are in line with the application.

Zoom Slider

The Zoom Slider can be used to zoom in the map by using the mouse. It consists of two buttons and a slider. The "+" button is intended for zoom in the map using a predefined change ratio. Zooming out using the same change ratio can be executed by clicking on the "-" button. The slider allows zoom the map in both directions using more detailed steps.

The zoom slider was integrated as a vertical gadget on the right border of the map in the WpfMap control and FormsMap control. So if a map control is added to the application, the zoom slider is visible by default. But this can be changed by using the ShowZoomSlider property of the WpfMap or ShowZoomSlider property of the FormsMap.

If a self-designed WPF map control is used, it can be customized to integrate the zoom slider at all and/or its location in the map. If the zoom slider should be integrated, the zoom slider gadget has to be added in design mode. There are no restrictions where to place it in the map's visible area.

Scale Gadget

The Scale Gadget shows the current scale of the map and can be found on the left bottom of the WpfMap and FormsMap. The scale can either be shown in km or in miles. To switch between these modes, the property UseMiles of the WpfMap or UseMiles of the FormsMap can be used.

To hide the scale, the properties ShowScale of the WpfMap and ShowScale of the FormsMap can be used. By default, the scale gadget is shown on the map. When zooming out of the map very much, the scale is no more shown. In this context a scale cannot be calculated for the whole map due to the underlying Mercator format. The deviation of the scales between the upper map border and the lower one, exceeds an acceptable limit.

In case of using a self-configured WPF map in the application, the scale gadget has to be explicitly added in the XAML file and it can be positioned at an arbitrary location in the map.

Coordinates Gadget

The coordinates gadget shows the current mouse position on the map in degrees. If the mouse is moved, the coordinates changes. These coordinates are shown centered on the bottom of the map.

Like the other map gadgets, the coordinates can also be hidden. To do so, the properties ShowCoordinates of the WpfMap and ShowCoordinates of the FormsMap can be used.

For a self-configured WPF map in the application, the location where to show the coordinates can be defined on your own, but the coordinates gadget has to be added explicitly in the XAML file.

Copyright Gadget

The copyright gadget is the only gadget which is fixed and cannot be removed. It shows the copyright information of the map in the right bottom corner of the map.

Layers Gadget

To show in a quick overview all layers which are contained in the map, the layers gadget can be used. This gadget can be expanded to show all layers with shortcuts to change some properties, and alternatively collapsed to save room on the map. The WpfMap and FormsMap show this gadget on the right at the bottom of the map window.

The check boxes and sliders in the layers gadget allows to

  • change the visibility of the layer.

  • change the selectability of the layer.

  • dim each layer separately from transparent to opaque.

Additionally, via drag and drop of the layers (click on the names) the layer order in the map can be changed.

Layers gadget in expanded state.
Layers gadget in collapsed state.

To hide the layers gadget completely, the property ShowLayers of the WpfMap or ShowLayers of the FormsMap can be used.

Using an own designed XAML for the WPF map, this gadget must be added explicitly to see it in the map.

Magnifier Map

PTV also offers a gadget showing a certain part of the map more detailed. It represents a magnifier tool which blows up a certain part of the map. The zoom factor can be changed at runtime. To start the magnifier, just press "m" on the keyboard and click in the map. After having released the mouse button, the magnifier can be moved via the mouse. The magnifier can be seen as long as the "m" key is pressed. By scrolling the mouse wheel, the zoom factor of the magnifier can be changed. The magnifier can be stopped by releasing the "m" key.

Magnifier on the map

To disable the magnifier, the property ShowMagnifier of the WpfMap or ShowMagnifier of the FormsMap can be used. In this case, nothing happens when pressing on the "m" key.

Clients building their own XAML file for the map have to add the magnifier gadget explicitly.

Navigation Gadget

By means of the navigation gadget it is possible to scroll in the map using the buttons of this gadget. The map can be scrolled up, down, right or left by pressing the corresponding buttons. The center button is reserved for future purposes and currently does not offer any functionality. The scroll distance is set to a fix value and the gadget is shown in the top left corner of the map.

Opaque navigation gadget.
Semi-transparent navigation gadget.

Like the other gadgets, the navigation gadget can also be hidden. For this purpose use the property ShowNavigation of the WpfMap or ShowNavigation of the FormsMap.

For a user defined WPF map based on the core map, the navigation gadget has to be added explicitly to be contained in the map.

Overview Map

Additionally to the map itself, a second small map called overview map is offered, which can be displayed in the above right corner of the WpfMap and FormsMap. This overview map shows the standard map in miniature display and a small rectangle on it visualizing the currently displayed map section. The overview map can be expanded and collapsed.

Overview map in expanded state.
Overview map in collapsed state.

To hide the overview map at all, the property ShowOverview of the WpfMap or ShowOverview of the FormsMap can be used.

To style an own map by building an own XAML file, the overview gadget has to be added explicitly to see it in the map.
