XServer Connection

This use case demonstrates how to connect to different types of PTV xServers, which provide the fundamental functionality used by the classes of the SDK and Demo Center. The PTV xServer types include a local hosted version, a previously available PTV hosted version, and the PTV xServer internet version in the Cloud.

This page contains the following sections:

  • Types of PTV xServer hostings
  • Checking URL and credentials
  • Usage of the PTV xServers
  • Use case manual
  • See Also
Types of PTV xServer hostings

For calculating routes, rendering maps or finding locations, PTV xServers are needed to fulfill these requirements. The PTV xServers can be hosted on-premise, by PTV Group or in the Cloud, currently provided by Microsoft Azure. Besides the URL needed for each of these hostings, the Cloud-based version requires additional credentials, i.e. user name and password, for granting access. Because this information allows complete access (especially billing) and is sent via HTTPS requests, a less powerful version is available: For user name 'xtok' can be used and for password the provided XToken, which is available after your account data is activated.

Checking URL and credentials

To prepare the application for PTV xServer usage, the SDK provided class XMapMetaInfo can be used to check the URL and optionally the credentials:

C#
Copy
var newXMapMetaInfo = new XMapMetaInfo(url);
newXMapMetaInfo.SetCredentials("xtok", xToken);
newXMapMetaInfo.CheckCredentials("xtok", xToken); // causes an exception if something goes wrong

If the URL or the credentials do not establish a valid PTV xServer connection, an exception is thrown. Otherwise, the XMapMetaInfo instance can be used for further configuration (see below).

Usage of the PTV xServers

After a successful establishment of a PTV xServer connection, a map can be shown with a base layer containing towns, streets and background areas (urban areas, woodland, etc.). Some additional layers are available and can be integrated into this map, at which the URL and (eventually) credentials mentioned above are needed. Therefore this information has to be provided for the layer creation:

C#
Copy
map.InsertPoiLayer(UseCase.ManagedAuthentication.XMapMetaInfo, "Poi", "default.points-of-interest", "Points of interest");
Use case manual

To test and use the PTV xServer configuration (at runtime), three predefined URLs are selectable in the use case expander, so the current PTV xServer can be changed at runtime:

PTV xServer Connection use case

Predefined PTV xServer hosts exists in the combobox of the use case expander, but own PTV xServer hostings can be defined too by entering the corresponding URL. The already predefined xToken in Demo Center application changes from time to time due to its limited validity.
